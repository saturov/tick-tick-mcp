# ticktick-skill

Набор CLI-команд для выгрузки данных из TickTick: проекты, все открытые задачи, открытые задачи без дат, выполненные задачи за дату.

## When to Use

Активируй скилл, когда пользователь просит:
- показать список проектов в TickTick
- выгрузить все открытые задачи (в том числе из project folder'ов)
- выгрузить открытые задачи (особенно без дат)
- выгрузить задачи без дедлайна в файл
- посмотреть выполненные задачи за сегодня / за дату
- получить сделанное за конкретный день
- ограничить выгрузку определёнными проектами
- спланировать рабочий день / неделю / период по задачам
- посмотреть просроченные задачи и решить что взять сегодня
- найти забытые задачи в backlog
- получить план дня с учётом встреч и приоритетов

**Первое действие при любом запросе:** проверь готовность окружения:

```bash
ls -la
python3 -c "import sys; print(sys.prefix)"
echo "${VIRTUAL_ENV:-no venv}"
```

Затем сообщи пользователю статус:
- Есть ли `.venv` в рабочей папке
- Активирован ли venv
- Установлены ли зависимости

## Setup

Перед первым использованием:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r <SKILL_SCRIPTS_DIR>/requirements.txt
pip install -e .
```

Где `<SKILL_SCRIPTS_DIR>` — путь к `skills/ticktick-skill/scripts/` относительно этого репозитория.
Основной доступ к TickTick API предоставляет локальный пакет `ticktick_mcp`; скилл хранит CLI-workflow, формат вывода и планирование.

Настрой `TICKTICK_API_KEY` (рекомендуется) или логин/пароль:

```bash
export TICKTICK_API_KEY='your_api_key'
```

Если `TICKTICK_API_KEY` задан, используется TickTick Open API (Bearer-токен) — быстрее и не требует логина/пароля. Если не задан — запрашивается интерактивный ввод логина/пароля через TickTick Web API.

## Режим 1: Список проектов (projects)

**Триггеры:** "покажи проекты", "список проектов", "какие проекты в тиктик", "список досок"

**Запуск:**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-projects
```

**Формат вывода:** `id<TAB>name`, по одному проекту на строку, отсортировано по имени.

**Что сообщить пользователю:** список проектов. Если проектов много — выведи всё, пользователь сам решит что нужно.

## Режим 2: Все открытые задачи (open-tasks)

**Триггеры:** "выгрузи все задачи", "покажи открытые задачи", "все задачи из проекта", "задачи из Рабочее", "открытые задачи с датами"

**Запуск (в stdout):**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-tasks-open
```

**Запуск (в файл):**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-tasks-open --output tasks.json
```

**Запуск с фильтром по проектам:**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-tasks-open --project "Work" --output tasks.json
```

**Запуск с фильтром по project folder (группе проектов):**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-tasks-open --project "Рабочее" --output tasks.json
```

**Параметры:**

| Параметр | Обязателен | По умолчанию | Описание |
|----------|-----------|--------------|----------|
| `--output` | нет | stdout | Путь к JSON-файлу для сохранения |
| `--project` | нет | все проекты | Имя или ID проекта, или имя project folder. Можно повторять. |

Флаг `--project` поддерживает точное совпадение имени (без учёта регистра) или ID проекта. Также поддерживает имена project folder'ов — при совпадении выгружаются задачи из всех проектов внутри folder'а.

**Формат вывода:** JSON-массив объектов с полями `raw` (все поля задачи из TickTick) и `meta` (время выгрузки, источник API).

**Что сообщить пользователю:**
- Сколько задач выгружено
- Куда сохранено (файл или stdout)
- Если указаны проекты или folder'ы — какие именно были отфильтрованы

<SKILL_SCRIPTS_DIR>/ticktick-tasks-undated --output snapshot.json

<SKILL_SCRIPTS_DIR>/ticktick-tasks-undated --project "Work" --project "Personal" --output snapshot.json

Флаг `--project` поддерживает точное совпадение имени (без учёта регистра) или ID проекта. При неоднозначном имени утилита попросит использовать ID.

**Формат вывода:** JSON-массив объектов с полями `raw` (все поля задачи из TickTick) и `meta` (время выгрузки, источник API).

## Режим 4: Выполненные задачи за дату (completed-tasks)

**Триггеры:** "что сделано за сегодня", "выполненные задачи за вчера", "сделанное за дату", "completed tasks за пятницу"

**Запуск (сегодня в stdout):**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-tasks-completed
```

**Запуск (конкретная дата в файл):**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-tasks-completed --date 2026-03-26 --output completed.json
```

**Запуск с фильтром по проектам:**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-tasks-completed --date 2026-03-26 --project "Work" --output completed.json
```

**Параметры:**

| Параметр | Обязателен | По умолчанию | Описание |
|----------|-----------|--------------|----------|
| `--date` | нет | сегодня | Дата в формате YYYY-MM-DD |
| `--output` | нет | stdout | Путь к JSON-файлу для сохранения |
| `--project` | нет | все проекты | Имя или ID проекта. Можно повторять. |

**Формат вывода:** JSON-массив объектов с полями `raw` (все поля задачи, включая `projectName`) и `meta` (`completed_at`, `fetched_at`, `source`). Задачи отсортированы по времени выполнения.

**Что сообщить пользователю:**
- Сколько выполненных задач найдено за дату
- Итоговый список задач с названиями и временем выполнения
- Куда сохранено (файл или stdout)

**Важно:** команда `ticktick-tasks-completed` работает **только через логин/пароль** (TickTick Web v2 API). Open API (`TICKTICK_API_KEY`) для неё не поддерживается. Если пользователь использует API key — сообщи ему, что для этой команды нужен логин/пароль.

## Режим 6: Обновление задачи (update)

**Триггеры:** "измени задачу", "поменяй дату", "обнови приоритет", "перенеси на завтра", "поставь дедлайн", "переименуй задачу"

**Требование:** команда работает **только через Open API** (`TICKTICK_API_KEY`). Логин/пароль не поддерживаются — обновление идёт через `POST /open/v1/task/{taskId}`.

**Запуск:**

```bash
<SKILL_SCRIPTS_DIR>/ticktick-task-update TASK_ID --due-date 2026-05-20
```

**Параметры:**

| Параметр | Обязателен | По умолчанию | Описание |
|----------|-----------|--------------|----------|
| `task_id` | да | — | ID задачи (позиционный) |
| `--start-date` | нет | — | Дата начала (YYYY-MM-DD или ISO 8601) |
| `--due-date` | нет | — | Дедлайн (YYYY-MM-DD или ISO 8601) |
| `--title` | нет | — | Новое название задачи |
| `--priority` | нет | — | Приоритет: 0=none, 1=low, 3=medium, 5=high |

**Примеры:**

```bash
# Перенести дедлайн на завтра
<SKILL_SCRIPTS_DIR>/ticktick-task-update 6a06c2b4b7c11102950356fb --due-date 2026-05-20

# Задать дату начала и дедлайн
<SKILL_SCRIPTS_DIR>/ticktick-task-update 699439446a82510f94fe9d33 --start-date 2026-05-19 --due-date 2026-05-21

# Изменить приоритет на высокий
<SKILL_SCRIPTS_DIR>/ticktick-task-update 699439446a82510f94fe9d33 --priority 5

# Переименовать задачу
<SKILL_SCRIPTS_DIR>/ticktick-task-update 699439446a82510f94fe9d33 --title "Новое название"
```

**Формат вывода:** JSON с обновлёнными полями задачи или с `id`.

**Что сообщить пользователю:**
- Какие поля были изменены
- Результат операции (успех / ошибка)

## Режим 7: Планирование дня/недели (planning)

**Это CLI-оркестрируемый режим.** Скрипт `ticktick-plan` выполняет сбор данных, timezone-aware классификацию, построение календаря и размещение задач. Агент интерпретирует результат и обсуждает правки с пользователем.

**Триггеры:**
- "спланируй мой рабочий день"
- "спланируй неделю"
- "посмотри, что у меня просрочено, и предложи, что взять сегодня"
- "найди забытые задачи в backlog"
- "у меня сегодня рабочий день с 11 до 19, спланируй"
- "не ставь большие задачи после 16:00"
- "оставь мне больше буфера между встречами"
- "сегодня хочу закрывать только мелкие хвосты"
- "найди одну большую задачу, которую лучше сделать в длинное окно"
- "проведи ревью задач без даты"
- "спланируй период с понедельника по среду"
- "посмотри, что у меня на этой неделе"

### Принцип работы

Скрипт делает всю механическую работу: классификацию, календарь, приоритизацию и размещение. Агент помогает пользователю уточнить параметры (часы, проекты, предпочтения) и интерпретирует результат.

### Шаг 1-2. Определить параметры

Понять из запроса пользователя:
- Период (по умолчанию today)
- Рабочие часы (по умолчанию 09:00–18:00)
- Обед — если пользователь явно указал время обеда, передать `--lunch-start` и `--lunch-end`. Если не указано — обед не учитывается (нет жёсткого дефолта).
- Фокусные проекты (--plan-project)
- Проекты исключения из fixed-блоков (--skip-fixed-project)

### Шаг 3. Запустить скрипт

```bash
<SKILL_SCRIPTS_DIR>/ticktick-plan --period today --work-start 10:30 --work-end 19:00
```

С обеденным перерывом:

```bash
<SKILL_SCRIPTS_DIR>/ticktick-plan --period today --work-start 10:30 --work-end 19:00 --lunch-start 13:00 --lunch-end 14:00
```

Или с фильтром по проектам:

```bash
<SKILL_SCRIPTS_DIR>/ticktick-plan --plan-project "Найм" --skip-fixed-project "Регулярное"
```

Для JSON-вывода (удобно для AI-анализа):

```bash
<SKILL_SCRIPTS_DIR>/ticktick-plan --json --output /tmp/plan.json
```

### Шаг 4-9. Скрипт делает автоматически
- Автоопределение таймзоны (из TickTick или локальной)
- Выгрузка всех открытых задач + фильтрация по --plan-project
- Timezone-aware классификация: fixed / dated_flexible / overdue / backlog
- Построение карты свободных окон с учётом working hours
- Эвристическая оценка длительности и типа задач
- Жадное размещение в окна с 20% буфером
- Детекция проблем: блокеры, перегрузка, много просрочки

### Шаг 10. Показать план пользователю

Вывод — markdown-план или JSON. Агент показывает пользователю и обсуждает правки.

### Шаг 11. Режим диалога

После показа черновика — жди правок пользователя. Типичные корректировки:
- "эту задачу не хочу сегодня"
- "это важнее"
- "после 16 я уже не смогу фокусироваться"
- "оставь больше буфера"
- "перенеси это на завтра"

Перезапусти скрипт с уточнёнными параметрами.

### Шаг 12. Что подсвечивать отдельно

Скрипт автоматически выводит предупреждения:
- Перегрузка дня (задач больше, чем свободного времени)
- Много просрочки (≥5 в одном проекте)
- Заблокированные задачи (содержат "жду"/"waiting")

Агент дополнительно обращает внимание на:
- Мутные задачи — предложить переформулировать
- Старый backlog — предложить чистку
- Нет фокусных окон — предложить перенести крупную работу

### Недельное планирование

```bash
<SKILL_SCRIPTS_DIR>/ticktick-plan --period week --work-start 10:30 --work-end 19:00
```

Скрипт анализирует каждый день недели. Агент дополнительно:
- Находит дни с большим количеством встреч (только короткие задачи)
- Находит дни с длинными фокусными окнами
- Распределяет просроченные задачи по неделе
- Предлагает дни для чистки backlog

### Параметры

| Параметр | Обязателен | По умолчанию | Описание |
|----------|-----------|--------------|----------|
| `--period` | нет | today | today, tomorrow, week, YYYY-MM-DD, YYYY-MM-DD YYYY-MM-DD |
| `--work-start` | нет | 09:00 | Начало рабочего дня (HH:MM) |
| `--work-end` | нет | 18:00 | Конец рабочего дня (HH:MM) |
| `--lunch-start` | нет | — | Начало обеда (HH:MM). Только когда задан явно |
| `--lunch-end` | нет | — | Конец обеда (HH:MM). Только когда задан явно |
| `--plan-project` | нет | все | Проект/папка для планирования (повторяемый) |
| `--skip-fixed-project` | нет | — | Исключить проект из fixed-блоков (повторяемый) |
| `--json` | нет | — | JSON вместо markdown |
| `--output` | нет | stdout | Сохранить в файл |
| `--timezone` | нет | авто | Переопределить таймзону |

**Важно:** скрипт не вносит изменения в TickTick. Агент может предложить `ticktick-task-update` для переноса задач, но только с подтверждения пользователя.

## Error Handling

| Ситуация | Действие |
|----------|----------|
| Нет `.venv` | Предложи создать: `python3 -m venv .venv && source .venv/bin/activate` |
| Venv не активирован | Напомни активировать: `source .venv/bin/activate` |
| Не установлен `ticktick-py` | Предложи: `pip install -r <SKILL_SCRIPTS_DIR>/requirements.txt` |
| Нет `TICKTICK_API_KEY` и нет логина | Для `projects` и `ticktick-tasks-undated`: предложи задать `TICKTICK_API_KEY` или ввести логин/пароль интерактивно. Для `ticktick-tasks-completed`: нужен логин/пароль |
| `auth failed` | Неверный логин/пароль или API key. Предложи проверить и попробовать снова |
| `sync failed` | Проблема сети или API TickTick. Предложи повторить позже |
| `project not found` | Указанный проект не найден. Предложи запустить режим 1 (список проектов) чтобы увидеть доступные |
| `project or folder not found` | Указанный проект или project folder не найден. Предложи запустить режим 1 (список проектов) чтобы увидеть доступные |
| `project name is ambiguous` | Несколько проектов с одинаковым именем. Покажи их ID и попроси уточнить |
| `task not found` | Задача с указанным ID не найдена. Проверь ID через `ticktick-tasks-open` |
| `TICKTICK_API_KEY is required for task updates` | Для `ticktick-task-update` нужен API-ключ. Логин/пароль не поддерживается |
| `at least one field to update is required` | Не указано ни одного поля для обновления. Добавь `--due-date`, `--title` или другой параметр |
| `invalid date` | Неверный формат даты. Укажи что нужен YYYY-MM-DD |
| Проект "Рабочие" не найден | Запусти режим 1 (`ticktick-projects`) и покажи доступные проекты. Возможно, folder называется иначе |
| Нет задач для планирования | Явно сообщи об этом. Пустой план — допустимый исход |
| Весь день занят встречами | Нормальная ситуация. Сообщи, что свободных окон нет, предложи перенести задачи на другой день |
